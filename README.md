# AI Brief Decoder Lite

Прототип платформенного решения для декодирования и анализа клиентских брифов с помощью ИИ.

## Структура проекта

- `backend/` — FastAPI сервис для обработки брифов, валидации вывода через Pydantic и сохранения результатов в PostgreSQL.
- `extension/` — Chrome-расширение на WXT + React + TS для взаимодействия с пользователем.
- `AI_USAGE.md` — Отчет об использовании ИИ-помощников в разработке.
- `provider_usage.md` — Инструкция по переключению и работе с провайдерами ИИ (Fake и Gemini) с учетом ретраев.
- `notes.md` — Заметки о принятых архитектурных решениях и компромиссах.

## Быстрый запуск бэкенда

### Требования
* Python 3.12+
* Poetry
* PostgreSQL (запущенный локально или через Docker Compose)

### Шаги запуска:

1. **Установка зависимостей**:
   ```bash
   cd backend
   poetry install
   ```

2. **Настройка окружения**:
   Создайте файл секретов `backend/config/.secrets.toml` (добавлен в `.gitignore`) для локальной разработки:
   ```toml
   GEMINI_API_KEY = "ваш-api-ключ-gemini"
   ```
   *Вы также можете задать системную переменную окружения `GEMINI_API_KEY` напрямую.*

3. **Запуск миграций базы данных**:
   Для применения схемы данных к локальной СУБД:
   ```bash
   poetry run alembic upgrade head
   ```

4. **Запуск сервера**:
   Вы можете запустить сервер разработки с помощью команды:
   ```bash
   make run
   ```
   Или напрямую через Poetry:
   ```bash
   poetry run uvicorn app.main:create_app --factory --reload --host 0.0.0.0 --port 8000
   ```
   Документация Swagger UI будет доступна по адресу: [http://localhost:8000/docs](http://localhost:8000/docs)

## Тестирование и линтинг

Все автоматизированные проверки можно выполнить с помощью команд из `Makefile`:

* **Запуск тестов**:
  ```bash
  make test
  ```
  *(Тестовая база данных `brief_decoder_test` создается и мигрируется автоматически)*
* **Форматирование кода**:
  ```bash
  make format
  ```
* **Проверка линтером**:
  ```bash
  make lint
  ```
* **Проверка типов**:
  ```bash
  make typecheck
  ```
