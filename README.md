# AI Brief Decoder Lite

Прототип платформенного решения для декодирования и анализа клиентских брифов с помощью ИИ.

## Структура проекта

- `backend/` — FastAPI сервис для обработки брифов, валидации вывода через Pydantic и сохранения результатов в PostgreSQL.
- `extention/` — Chrome-расширение на WXT + React + TS для взаимодействия с пользователем.
- `AI_USAGE.md` — Отчет об использовании ИИ-помощников в разработке.
- `provider_usage.md` — Инструкция по переключению и работе с провайдерами ИИ (Fake и Gemini) с учетом ретраев.
- `notes.md` — Заметки о принятых архитектурных решениях и компромиссах.
- `Makefile` — Корневой файл автоматизации для управления и запуска бэкенда и расширения из единой точки.

## Быстрый запуск

### Требования
* Python 3.12+ (для бэкенда)
* Node.js v18+ (для расширения)
* Poetry
* PostgreSQL (запущенный локально или через Docker Compose)

### 1. Установка всех зависимостей
Из корня проекта выполните команду для автоматической установки зависимостей бэкенда и расширения:
```bash
make install-all
```

### 2. Настройка окружения
Создайте файл секретов `backend/config/.secrets.toml` (добавлен в `.gitignore`) для локальной разработки:
```toml
GEMINI_API_KEY = "ваш-api-ключ-gemini"
```
*Вы также можете задать системную переменную окружения `GEMINI_API_KEY` напрямую.*

### 3. Запуск базы данных
Если у вас нет локального инстанса PostgreSQL, запустите базу данных в Docker:
```bash
make db-up
```

### 4. Запуск миграций базы данных
Примените схему данных к базе данных:
```bash
make migrate-back
```

### 5. Запуск бэкенда
Запустите сервер разработки FastAPI:
```bash
make run-back
```
Документация Swagger UI будет доступна по адресу: [http://localhost:8000/docs](http://localhost:8000/docs)

### 6. Запуск и сборка расширения (Frontend)
Для запуска расширения в режиме разработки (с поддержкой горячей перезагрузки HMR):
```bash
make run-ext
```
Для сборки расширения под Chrome MV3:
```bash
make build-ext
```
После сборки готовое к установке расширение находится в папке `extention/.output/chrome-mv3` и загружается в Chrome через страницу `chrome://extensions/` (с включенным «Режимом разработчика»).

## Тестирование, линтинг и проверки типов

Все автоматизированные проверки запускаются из корня проекта:

* **Запуск тестов бэкенда**:
  ```bash
  make test-back
  ```
* **Форматирование кода бэкенда**:
  ```bash
  make format-back
  ```
* **Проверка линтером бэкенда**:
  ```bash
  make lint-back
  ```
* **Проверка типов бэкенда**:
  ```bash
  make typecheck-back
  ```
* **Проверка типов расширения**:
  ```bash
  make typecheck-ext
  ```
* **Проверка линтером расширения**:
  ```bash
  make lint-ext
  ```
