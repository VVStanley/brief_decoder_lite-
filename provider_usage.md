# Инструкция по работе с провайдерами ИИ (provider_usage.md)

Проект поддерживает работу в двух режимах: с реальным ИИ (Gemini Flash) и с фейковым провайдером (заглушкой).

## 1. Фейковый провайдер (Fake Provider)

Используется по умолчанию для локального тестирования без API-ключей.

### Настройка
В конфигурационном файле бэкенда (`settings.toml` или переменные окружения) установите, убедитесь что в `.secrets.toml` настройка не переопределена:
```bash
LLM_PROVIDER=fake
```

### Поведение
При запросе декодирования бэкенд возвращает валидные заготовленные структуры данных. Сетевые запросы к внешним провайдерам не выполняются.

---

## 2. Реальный провайдер Gemini (Gemini Provider)

Использует PydanticAI и API Gemini Flash для разбора брифов.

### Настройка
1. Получите API-ключ Gemini.
2. В конфигурационном файле бэкенда установите:
```bash
LLM_PROVIDER=gemini
GEMINI_API_KEY=ваш_ключ
```

### 3. Настройка параметров надежности (Ретраи и Таймауты)

Поведение сетевых запросов ко всем провайдерам (включая Gemini) регулируется в `settings.toml` или через переменные окружения:

* **`LLM_MAX_ATTEMPTS`** (по умолчанию `3`) — максимальное количество попыток выполнения запроса к ИИ-модели при возникновении временных сетевых ошибок.
* **`LLM_TIMEOUT`** (по умолчанию `45.0`) — максимальное время в секундах, отводимое на выполнение одного запроса к API.

Эти параметры автоматически применяются декоратором `RetryingLLMProvider`, повышая общую стабильность обработки фоновых задач.
